## Основные изменения и улучшения:
1.	Product: Добавлен класс товара.
2.	OrderCollection: Заказы разных типов (дженерики) сложно хранить в одном списке List<Order<...>>. Поэтому введен базовый не дженерик класс OrderBase, от которого наследуется дженерик Order<T>. Это позволяет хранить все заказы в одной коллекции.
3.	Инкапсуляция: Поля заменены на Свойства (Properties) с protected set или private set, где это необходимо, чтобы нельзя было случайно сломать данные извне.
4.	Полиморфизм: В классе Delivery появились виртуальные методы (например, расчет стоимости), которые переопределяются в наследниках.

## Разбор архитектуры:
# 1.	Наследование Delivery:
*	Мы вынесли общие поля (Address, Phone) в абстрактный класс.
*	Методы CalculateShippingCost и GetEstimatedDeliveryDate сделаны virtual. Это позволяет каждому типу доставки иметь свою математику (например, самовывоз бесплатный, курьер дорогой, но дешевеет при большой сумме заказа).
# 2.	Дженерик Order<TDelivery> и OrderBase:
*	Это ключевой момент. Если бы мы оставили только Order<TDelivery>, мы бы не смогли создать List<Order>, т.к. Order<HomeDelivery> и Order<ShopDelivery> — это абсолютно разные типы данных для компилятора.
*	Создав абстрактный OrderBase, мы получили возможность работать со списком разнородных заказов (например, для админской панели), при этом сохраняя строгую типизацию внутри конкретного заказа (поле Delivery имеет конкретный тип, а не абстрактный).
# 3.	Реалистичность (Пункты 4.1 - 4.3):
*	HomeDelivery: Хранит имя курьерской службы и временной слот.
*	PickPointDelivery: Хранит ID постамата и компанию-провайдера.
*	ShopDelivery: Хранит ID магазина и часы работы, игнорирует внешние курьерские службы.
# 4.	Товары:
*	Теперь заказ содержит список List<Product>, а не абстрактную структуру. Сумма заказа рассчитывается динамически через LINQ (Products.Sum).
